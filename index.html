ECMAScript 6 features (1)

Posted on 2018-01-28 | In Article

const

Support for constants, variables which cannot be re-assigned new content. Notice: this only makes the variable itself immutable, not its assigned content (for instance, in case the content is an object, this means the object itself can still be altered).

let

Block-scoped variables (and constants) without hoisting.

‘let’ only validates in scope:

Temporal dead zone (TDZ)
If a variable is declared by ‘let’ in a block scope, it’s binding to the block scope and refrain from outside.

Repeat declaration is not allowed

ES6 has 6 methods of variable declaration

ES5 has two methods of variable declaration: var, function;
ES6 has 4 more methods: let, const, import, class.
